# Looker Studioで再現する場合の構成

## データソース構成

各媒体APIのデータは一度BigQuery（またはスプレッドシート）に日次でETLし、
Looker Studioからは **非正規化済みのビュー** を参照する構成にする。

```
Meta / TikTok / SmartNews / Popin / Taboola / Outbrain API
        │（日次バッチ / Cloud Functions・GAS等）
        ▼
   BigQuery: raw_ad_metrics_daily
        │（dbt等でJOIN・集計）
        ▼
   BigQuery View: v_team_kpi / v_project_kpi / v_creative_analysis
        │
        ▼
   Looker Studio データソース（コネクタ: BigQuery）
```

## 主要ビュー例

- `v_team_kpi`：日次 × 案件 × Cost/CV/CPA（① チーム全体KPI進捗用）
- `v_project_kpi`：日次 × 案件 × キャンペーン/広告セット/広告 × 各指標（② 案件KPI進捗用）
- `v_creative_analysis`：日次 × クリエイティブ × 媒体/サイズ/種別 × 各指標（③ クリエイティブ分析用）

## 画面ごとの実装方法

### ① チーム全体KPI進捗
- KPIカード → **スコアカード**（総Cost, 総CV, 平均CPA など）＋条件書式でCPA超過を赤字表示
- 案件一覧 → **表（テーブル）** + 条件付き書式（CPA > 許容CPAで赤背景）
- 折れ線グラフ → **期間比較の時系列グラフ**（ディメンション: date、指標: Cost）
- 改善が必要な案件 → 案件一覧テーブルに **フィルタ（CPA超過 or 消化率>95%）** を適用した2つ目のテーブル

### ② 案件KPI進捗
- 案件セレクタ → **プルダウンフィルタ（案件名）**
- 期間指定 → Looker Studio標準の **期間コントロール**
- 3タブ構成 → Looker Studioの **ページ切り替え（キャンペーン/広告セット/広告の3ページ）** で代替
- サマリーカード → スコアカードを横並びで配置
- 折れ線グラフの指標切替 → **グラフ内のディメンション/指標コントロール**、または指標ごとに複数グラフを用意しページ内タブで切替
- CPAランキングTOP5 → 表（テーブル）を並び替え・行数制限（Top N）

### ③ クリエイティブ分析
- ランキング → 表（並び替え + 行数制限）
- 媒体別／静止画動画／サイズ別比較 → **棒グラフ**（ディメイジョン: media / type / size）
- 画像一覧 → Looker Studioの標準機能ではモーダル表示は不可のため、
  **画像付きテーブル（サムネイル列）** で一覧化し、クリック詳細はダッシュボード内の別ページ
  （案件・クリエイティブ名でフィルタしたドリルダウンページ）で代替する。

### ④ クリエイティブ入稿依頼（カンバン）
- Looker StudioはKanban UIを持たないため、この画面のみ **将来のReact化を前提に、
  現段階ではスプレッドシート + 簡易なステータス別ビュー（テーブルをステータスでフィルタしたタブ）**
  で代替するか、Notion/Asana等の既存ツールと連携する運用でカバーする。

### ⑤ 権限・設定
- Looker Studioの **共有権限（表示専用リンク/編集者）** をそのままロール分けに利用。
  - admin: 編集者
  - operator: 編集者（自分の担当案件フィルタ付きレポートを個別配布）
  - client: 閲覧者（フィルタ固定のレポートリンクを共有）

## フィルタ設計

Looker Studio共通のページレベルフィルタとして以下を用意し、React移行後もクエリパラメータとして
そのまま踏襲できるようにする。

- 案件（project_id）
- 媒体（media）
- 担当者（owner_id）
- 期間（date range）
- クリエイティブ名（部分一致検索）

## 移行時の互換性を保つポイント

1. Looker Studio用ビューのカラム名を、そのままAPIレスポンスのフィールド名（camelCase変換のみ）に合わせる。
2. スコアカード・テーブル・グラフ・フィルタの単位を、React側のコンポーネント（KpiCard / DataTable /
   LineChartCard / フィルタバー）に1:1で対応させる。
3. カンバン画面のみLooker Studioでは再現しないため、先行してReact + Next.js側で実装し、
   他画面がLooker Studioの間もこの画面だけ独立して先行運用できるようにする。
